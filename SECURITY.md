# Security Policy

OmniSaveSwitch is a hobby project maintained in spare time. There is no security team, no SLA, and no private disclosure channel.

## Reporting a vulnerability

Open a GitHub Issue and add the `security` label. Describe the vulnerability clearly. Do not include exploit code in the initial report — a description of the attack surface and impact is enough to start.

Issues labeled `security` are reviewed when time allows. There is no guaranteed response time.

## Scope

This project runs on a Nintendo Switch console and communicates with a local OmniSave server on your home network. It does not handle credentials, personal data, or network-accessible services beyond the local LAN sync protocol.

## No CVE process

This project is not registered with any CVE authority and does not issue CVEs. If a vulnerability is fixed, it will be noted in the CHANGELOG under the relevant release.
